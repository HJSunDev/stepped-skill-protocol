# Review Rubric

Use this rubric when the user does not provide a more specific standard.

Review dimensions:

- product essence: Does the artifact solve the real user problem?
- architecture fit: Are boundaries, responsibilities, and abstractions coherent?
- usability: Can the intended user understand and apply it?
- maintainability: Will future contributors be able to evolve it safely?
- evidence quality: Are claims grounded in observable artifacts?
- risk: What can fail, and how severe would the failure be?
- adoption path: Is there a realistic path from current state to useful adoption?

Evidence discipline:

- cite concrete artifact locations when possible;
- label inference as inference;
- do not invent missing evidence;
- do not turn preferences into findings without impact.
