# Step: Recommend Actions

## Objective

Turn evaluated findings into actionable recommendations.

## Resources

- `references/severity-scale.md`

## Instructions

1. Use the evaluation handoff as inherited state.
2. For each material finding, propose one or more practical actions.
3. Prioritize actions by severity, expected impact, and implementation effort.
4. Avoid generic advice. Each recommendation must connect to a finding and evidence.
5. Identify what should be done now, later, or not at all.
6. Do not write the final report yet.

## Output

Produce:

- action list;
- priority by action;
- expected impact;
- estimated effort or uncertainty;
- final-report outline.

## Completion Criteria

This step is complete when the final-report step can write a clean user-facing review.

## Handoff

Carry forward:

- prioritized findings;
- recommended actions;
- impact and effort;
- actions to defer or reject;
- final-report outline;
- residual risk.

Before reading `Next`, record an `SSP Handoff` block with those items in the execution context.
